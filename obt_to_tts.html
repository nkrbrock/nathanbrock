Oral Bible Translation to Text-to-Speech

Overview

Find code here.

The purpose of this project is to convert an audio Bible along with its written transcript into a Coqui-TTS system. This task can be split up into 4 main areas:

  • Preprocessing. This was by far the most complicated part of the process to figure out. In order to automate this process with ClearML, preprocessing needed to be broken down into multiple steps:
    • Download and extract the audio files that the user needs into a designated folder.
    • Split the biblical text into chapters and assign them names that align with MFA standards.
    • Create a phonetic dictionary text file that contains word-pronunciation pairs with each sound separated as required by MFA. ie calm k a l m
    • Clean up the audio and place it in the same folder as the split biblical text.
  • Montreal Forced Aligner
  • Convert data for Coqui
    • Takes the .textgrid files that MFA produced, and extracts the phonemic data to create a csv file that has one line per audio file title with its transcript. i.e. GEN_001.wav|ni...
  • Coqui-TTS
    • This uses the metadata.csv along with the audio file to train a Coqui-TTS model. Current model is GlowTTS, but may change in the future depending on results.
